# Gameplay Prototyping

Use this skill when the user has a game idea but not yet a crisp prototype definition.

## Workflow

- Identify the core player verb and the shortest loop that proves the idea is fun.
- Cut anything that does not support the first playable within the stated time budget.
- Define the minimum set of scenes, mechanics, UI, and assets required.
- Separate prototype blockers from stretch goals.
- End with a milestone list that another engineer or agent can execute directly.

## Output checklist

- One-sentence prototype goal
- Core loop in 3-5 steps
- In-scope systems
- Deferred systems
- Asset checklist
- Acceptance criteria for the first playable
