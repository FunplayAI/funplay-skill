Tell the user you are using `using-funplay-skills` to route the task safely.

Then:

1. Determine whether the project is Unity, Godot, or Cocos Creator.
2. Route commercial Unity mobile/F2P planning or production-readiness audits to `commercial-unity-game`.
3. Route Unity projects connected to Funplay MCP to `unity-mcp-workflow` when live editor readback, compilation, Play Mode, screenshots, or console logs are needed.
4. Route Cocos Creator page/popup canvas, UI retrofit, Workbench asset replacement, or minigame subpackage work to the matching Cocos engine skill.
5. For Unity without MCP, Godot, or unsupported Cocos work, say there is no verified FunPlay skill currently available and continue with normal cautious repository inspection.
6. Ask the user to confirm high-risk editor-generated files before broad edits.
7. End with a short verification checklist for the chosen engine.
