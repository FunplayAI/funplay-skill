description: "Choose the right engine-facing workflow before editing project files"
---

Tell the user you are using `using-funplay-skills` to route the task to the correct engine workflow.

Then:

1. Ask which engine or project type is in scope if it is not already obvious.
2. Use `commercial-unity-game` when the request is about commercial Unity mobile/F2P planning, production readiness, economy, retention, IAP/shop, data pipelines, or auditing whether a prototype can become a shippable product.
3. Use `unity-mcp-workflow` when a Unity project is connected to Funplay MCP and live editor verification is needed.
4. Use the Cocos engine skills for Cocos page/popup canvas entries, safe node removal, UI node retrofits, Workbench asset replacement, and minigame subpackage rules.
5. For unsupported engine work, say there is no verified FunPlay skill currently available and continue with normal project-specific inspection.
